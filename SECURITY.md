# Security policy

Do not post credentials, Pi authentication files, private session transcripts, source code, or verbose RPC diagnostic bundles in public issues.

For a security-sensitive report, contact the repository owner privately through GitHub. Include the affected PiTTy version, platform, reproduction steps, and the minimum redacted evidence needed to verify the problem.

PiTTy launches Pi and installed Pi extensions with the user's normal system permissions. Review third-party Pi packages before installing them.
